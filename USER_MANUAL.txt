User manual — Graph Movie Recommender
======================================

Purpose
-------
You pick movies you enjoy; the program suggests similar titles using a weighted graph of movies. On **MovieLens 20M**, edges use **genre overlap + rating correlation** (no cast in the core CSVs). On the small toy dataset, optional **movie_people.csv** adds cast and director. Recommendations use **multi-source Dijkstra** (shortest-path ranking). **BFS** and **DFS** show traversal order from one start movie.

Requirements
------------
Java 17 or newer. A graphical desktop (Swing + FlatLaf). At runtime **no network** is required once data files are local.

Full MovieLens 20M (Kaggle)
---------------------------
1. `python3 -m pip install -r scripts/requirements-movielens.txt` (use `-m pip` so packages install for the same `python3` that runs the script)
2. `python3 scripts/download_movielens.py`  
   This uses **kagglehub** to download **grouplens/movielens-20m-dataset** and copies `movies.csv` and `ratings.csv` into `data/movielens-20m/`.
3. Start the app from the project root. The first run **streams all ratings** (can take a few minutes) and then **builds the sparse graph** (can take several more minutes). A loading window shows progress. Later runs reuse `data/movielens-20m/cache/graph_cache.ser` for a faster start (ratings are still re-sampled once per launch for correlation).

Alternatively set **MOVIELENS_20M_DIR** to an absolute path that already contains `movies.csv` and `ratings.csv`.

How to start the program
-------------------------
1. `cd` to the project root (folder with `pom.xml`, `data/`, `run.sh`).
2. `./run.sh`  or  `mvn compile exec:java`
3. Optional argument: config directory (default `data`). The app prefers `data/movielens-20m/` when present.

Main window (updated layout)
----------------------------
• **Left — Movie list:** search by title or id, **Apply**, then Ctrl/Cmd-click several movies you like.  
• **Right — Output:** scrollable log of recommendations and traversals.  
• **Get recommendations:** Top K (1–100) ranked by shortest-path distance (lower = closer in the graph).  
• **Show BFS order / Show DFS order:** use the **first** selected movie as the start node (up to 50 steps).  
• **Reload dataset & rebuild graph:** ignores cache; slow on MovieLens 20M.

Small toy dataset (optional)
----------------------------
If `data/movielens-20m/` is absent, the app uses `data/movies.csv`, `data/ratings.csv`, and optional `data/movie_people.csv`, with cache at `data/cache/graph_cache.ser`.

Troubleshooting
---------------
• **Kaggle / kagglehub errors:** configure Kaggle credentials as required by your environment.  
• **OutOfMemory:** increase heap (`java -Xmx4g …`) or lower the per-movie rating cap in `RatingsReservoir` / neighbor counts in `SparseMovieGraphBuilder`.  
• **No recommendations:** try different seeds or delete the cache file and reload after tuning thresholds.
