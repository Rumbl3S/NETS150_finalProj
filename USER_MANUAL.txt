User Manual - Graph Movie Recommender
=====================================

Purpose
-------
You pick movies you enjoy, and the program suggests similar titles using a
weighted graph of movies. Edges combine genre overlap and Pearson correlation
on shared user ratings (with optional cast and director similarity in toy
mode). Recommendations come from multi-source Dijkstra (shortest-path
ranking), BFS and DFS show traversal order from a chosen start movie, and the
Visualize Graph button opens an interactive panel that draws the picks and
their neighborhood.

Requirements
------------
- Java 17 or newer
- Maven (only needed if running with `mvn compile exec:java`)
- A graphical desktop environment (Swing + FlatLaf)
- No network access is required at runtime once the data files are local.

How to start the program
------------------------
1. cd into the project root (the folder containing pom.xml, data/, and run.sh).
2. Run either:
     ./run.sh
   or
     mvn compile exec:java
3. An optional first argument sets the config directory (default is "data").
   The app prefers data/movielens-20m/ if present, otherwise it falls back to
   the bundled toy CSVs in data/.

Main window
-----------
The app uses a card-based layout (see launch.png):

- "How to Use" card: a quick five-step reminder of how to drive the UI.

- "Browse Movies" card: type a title or movie id in the search box and click
  Search. Select one or more rows in the list and click "Add to my picks" to
  seed the recommender.

- "Your Picks (Seeds)" card: stays put even when you change the search filter.
  Use "Remove selected" or "Clear all" to manage the seed set.

- "Output" card (right side): a scrollable log that shows the active dataset,
  recommendation results, BFS/DFS output, and any status messages.

- "Recommendations" card: choose how many recommendations to return (1-100)
  and click "Get recommendations". Results are ranked by shortest-path
  distance; lower distance means closer in the graph.

- "Graph Traversal (Class Requirement)" card: three buttons.
    Show BFS order   - BFS from the first movie in Your Picks.
    Show DFS order   - DFS from the first movie in Your Picks.
    Visualize Graph  - opens an interactive Swing window with the picks drawn
                       in green and their immediate neighbors in blue. Drag
                       to pan, scroll to zoom.

- "Reload dataset & rebuild graph": ignores any cached graph and rebuilds from
  scratch. This is slow on MovieLens 20M and is mainly useful after changing
  data files or tuning thresholds.

Using the full MovieLens 20M dataset
------------------------------------
1. Install Python deps:
     python3 -m pip install -r scripts/requirements-movielens.txt
2. Download the dataset (uses kagglehub to fetch grouplens/movielens-20m-dataset):
     python3 scripts/download_movielens.py
   This writes movies.csv and ratings.csv into data/movielens-20m/.
3. Start the app. By default the Java side caps reading at 1,000,000 rating
   rows for speed; set -Dmovielens.ratingLines=0 to read the entire file. The
   first run streams ratings, builds the sparse graph, and writes a cache file
   under data/movielens-20m/cache/graph_cache.ser. Later runs reuse that cache.

You can also point the app at any prepared folder by setting the environment
variable MOVIELENS_20M_DIR to an absolute path containing movies.csv and
ratings.csv.

Toy dataset (fallback)
----------------------
If data/movielens-20m/ is absent, the app loads data/movies.csv,
data/ratings.csv, and the optional data/movie_people.csv. The cache lives at
data/cache/graph_cache.ser.

Troubleshooting
---------------
- Kaggle / kagglehub errors: configure your Kaggle credentials per the
  kagglehub documentation.
- OutOfMemoryError: increase the heap (java -Xmx4g ...) or lower the per-movie
  rating cap in RatingsReservoir / the neighbor count in
  SparseMovieGraphBuilder.
- No recommendations returned: try different seeds, or delete the cache file
  (graph_cache.ser) and reload after adjusting similarity thresholds.
