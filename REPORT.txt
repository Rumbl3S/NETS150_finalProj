Short report — graph model, algorithms, example outputs
=========================================================

Graph model
-----------
Nodes are movies (integer ids aligned with the MovieLens-style `movies.csv`). Undirected weighted edges connect movie pairs whose combined similarity score is at least a fixed minimum. Similarity blends four signals with fixed weights: Jaccard overlap on genre tags, Jaccard overlap on actor names (from `movie_people.csv`), an indicator for same director, and a rating-correlation term. For the rating term we take users who rated both films; if at least two users overlap, we compute Pearson correlation between the two rating vectors and map the correlation from [-1,1] to [0,1]. The edge weight used for shortest-path search is (1 − similarity), so more similar neighbors are linked by cheaper edges and appear closer under Dijkstra.

Algorithms
----------
• Breadth-first search (BFS) and depth-first search (DFS) traverse the adjacency list in deterministic ascending neighbor id order; they ignore edge weights and are intended to illustrate classic graph exploration from a start movie.
• Recommendations use multi-source Dijkstra: every selected “liked” movie is a source with initial distance zero; the algorithm relaxes edges in best-first order until shortest-path distances are known. Movies are ranked by increasing distance from the seed set (excluding the seeds themselves).

Example outputs (illustrative; your numeric distances may differ slightly)
-------------------------------------------------------------------------
If you select “Heat (1995)” and click Recommend, you should see other crime/action titles that share cast or rating patterns with high similarity, such as “Casino (1995)” or “Get Shorty (1995)”, appearing near the top with smaller distances than unrelated family films.

If you select “Toy Story (1995)” and run BFS from that row, the printed order begins at Toy Story and expands ring-by-ring to adjacent animation/adventure neighbors like “Balto (1995)” before deeper branches.

If you select “Twelve Monkeys (1995)” and run DFS, the preorder listing follows one branch of the similarity graph until the node cap (40) is reached, then unwinds according to the depth-first policy.
