Dataset source and layout
=========================

Full MovieLens 20M (recommended)
--------------------------------
1. Install Python deps (use the same interpreter as `python3`):  
   python3 -m pip install -r scripts/requirements-movielens.txt
2. From the project root run:  python3 scripts/download_movielens.py

This uses kagglehub to download **grouplens/movielens-20m-dataset** from Kaggle and copies **movies.csv** and **ratings.csv** into:

  data/movielens-20m/

The Java app resolves datasets in this order:
  • Environment variable **MOVIELENS_20M_DIR** (absolute path to a folder that already contains movies.csv + ratings.csv), else
  • **<config>/movielens-20m/** under your config directory (default config is the `data` folder, so `data/movielens-20m/`), else
  • **<config>/movies.csv** and **<config>/ratings.csv** (small bundled sample) plus optional **movie_people.csv**.

Official GroupLens page (citation and other releases):
  https://grouplens.org/datasets/movielens/

Kaggle mirror used by the script:
  https://www.kaggle.com/datasets/grouplens/movielens-20m-dataset

Processing note
---------------
The full ratings file is streamed once; each movie keeps a **reservoir sample** of user ratings (default 400) for correlation. The graph is **sparse** (top similar neighbors per movie) so the program stays responsive on ~27k movies and ~20M ratings.

Cache
-----
After the first successful build, the graph is saved under **<dataset-folder>/cache/graph_cache.ser**. Delete that file to force a rebuild.
